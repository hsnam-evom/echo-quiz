# 에코 퀴-즈 (Echo Quiz) — 기획안 v1

심초음파(Echocardiography) 임상 지식을 레트로 게임 형식으로 풀어보는 퀴즈 웹앱.

---

## 1. 데이터 소스

`echo_quiz_100.xlsx` → 빌드 시 `questions.json`으로 변환하여 번들.

| 필드 | 설명 |
|------|------|
| `id` | 문항 번호 (1~100) |
| `type` | 유형코드 (A1~H4) |
| `category` | 분류명 (예: "측정 원리", "유사 질환 감별") |
| `question` | 질문 |
| `answer` | 정답 텍스트 |
| `distractors` | 오답 3개 배열 |
| `explanation` | 해설 (가이드라인 근거) |

- 100문항, 결측치 없음. 8개 대분류(A 측정원리 ~ H 적응증), 26개 세부유형.
- ⚠️ 정답/해설이 클라이언트 번들에 포함됨 → 개발자도구로 들여다보면 "컨닝" 가능. 재미 위주 퀴즈라 허용 범위로 판단(서버 채점은 과한 오버엔지니어링). 추후 필요 시 서버 채점으로 격상 가능.

## 2. 게임 플로우 (상태 머신)

```
START ──닉네임 입력──▶ PLAYING(Q1..Q10) ──▶ RESULT ──재도전──▶ START
                              │
                        각 문항 처리 루프 ▼
```

**문항 처리 루프 (문항당):**
1. 질문 + 4지선다 표시. **정답 위치는 매 렌더마다 랜덤 셔플** (같은 문항이어도 정답 번호가 매번 바뀜).
2. 1차 시도
   - 정답 → ✅ 정답 처리 (만점), 해설 표시
   - 오답 → 고른 보기 비활성화/❌ 표시, **"한 번 더 기회!"** 안내, 남은 보기에서 재선택
3. 2차 시도
   - 정답 → 🟡 정답 처리 (부분점수), 해설 표시
   - 오답 → ❌ 오답 처리 (0점), **정답 강조** + 해설 표시
4. "다음 문제" → 다음 문항 / 마지막이면 RESULT

**출제 규칙:** 100문항 중 **랜덤 10문항** (중복 없이 셔플 후 앞 10개).

## 3. 점수 체계 (제안)

| 결과 | 점수 |
|------|------|
| 1차 정답 | **10점** |
| 2차 정답 | **5점** |
| 2번 다 틀림 | 0점 |

- 만점 100점. 결과 화면 + 리더보드에 합산 점수 기록.
- (대안: 단순 정답수 카운트 = 맞으면 1, 틀리면 0. → 아래 결정 필요)

## 4. 결과 화면

- 최종 점수 (예: **75 / 100**) + 정답/부분/오답 개수.
- **10문항 결과 요약 리스트**: 각 문항 질문 / 내 결과(✅🟡❌) / 정답 / 해설 펼쳐보기.
- **점수별 재미 문구** (티어별, 초안 — 톤 조절 가능):

| 점수대 | 문구 (높을수록 위, 이모지 앞·뒤 1개씩) |
|--------|------|
| 90~100 | 👑 에코의 신, 에코 그 자체, 살아있는 에코 🫀 |
| 70~89 | ❤️ 당신은 에코밖에 모르는 에코바보 ❤️ |
| 50~69 | 🩺 Mild 에코지식 이상 소견이 관찰됩니다. 지속적인 추적 관찰이 필요합니다. 👀 |
| 25~49 | ⚠️ Moderate 에코지식 이상 소견이 관찰됩니다. 즉시 가이드라인을 방문하세요 📖 |
| 0~24 | 🚨 혈류가 뇌까지 안간듯 합니다. 신속한 cardiac output 확인이 필요합니다. 🩸 |

- 액션 버튼: **재도전**(닉네임 다시 입력) / **리더보드 보기**.

## 5. 리더보드

- 게임 종료 시 `{닉네임, 점수, 시각}` 기록.
- **상위 10명** 표시 (점수 내림차순, 동점 시 먼저 달성한 순).
- 저장 방식 — 아래 결정 필요:
  - **(A) 글로벌 공유 리더보드 (추천)**: Vercel Serverless Function + Upstash Redis(Vercel Marketplace 무료 티어). 모든 방문자가 같은 순위표 공유. Redis Sorted Set(`ZADD`/`ZREVRANGE`)으로 구현.
    - `GET /api/leaderboard` → 상위 10
    - `POST /api/score` → 점수 제출 (닉네임 길이/문자 검증, 기본 욕설 필터)
  - **(B) 로컬 전용**: `localStorage` 저장. 인프라 0, 단 기기/브라우저별 개별 순위 (공유 안 됨).

## 6. 비주얼 — 레트로 스타일

- **CRT/아케이드 감성**: 스캔라인 오버레이, 픽셀 폰트(`Press Start 2P` + 한글은 `DungGeunMo`/`Galmuri` 등 픽셀 한글 폰트), 네온 그린/마젠타 팔레트, 깜빡이는 커서, 8비트 효과음(정답/오답/클리어, 음소거 토글).
- 반응형(모바일 우선). 과한 모션은 `prefers-reduced-motion` 존중.

---

## 7. 기술 스택 (제안)

- **프론트엔드**: 바닐라 JS SPA (의존성 최소, 빠른 로드) + 레트로 CSS. *(대안: Next.js/React — 결정 필요)*
- **백엔드**: 리더보드 (A)안 선택 시 Vercel Serverless Functions(`/api`) + Upstash Redis.
- **빌드**: Excel→JSON 변환 스크립트(`scripts/build-questions.mjs`, 빌드 시 1회 실행).
- **배포**: 기존 `echo-quiz` GitHub 저장소 + Vercel 프로덕션(`echo-quiz.vercel.app`). git push 자동배포 연동도 함께 설정.
